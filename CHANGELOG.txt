  unresponsive.


-------------------------------------------------------------------------------
NetConf Tool  1.2.6 -- 2024/4/10
-------------------------------------------------------------------------------

**Changes**

- Tweak log show. Use different colors according to different logs.

- More accurate recording of rpc time, from the time before sending the socket
  to the time when the rpc reply is received.

- Removing XML nodes that contain only whitespace characters in their text,
  which are referred to as 'select nodes'.

  According to RFC4714, nodes must contain non-whitespace characters.

- Adds sequential execution control to allow only sequential execution. A new
  RPC is not allowed to be sent without receiving the response to the previous
  RPC.

NetConf Tool  1.2.2 -- 2024/3/5
-------------------------------------------------------------------------------

  reply, thereby improving UI response speed.

  'Copy Content to Clipboard' menu, enabling users to view the complete reply.

**Changes**

- Change 'Session History' selection mode.

- Improve the accuracy of RPC execution time calculation.

- The message-id in the rpc becomes an incrementing sequence number, and the
  sequence number starts at 0 when the session is connected or reconnected.

- All RPC requests are add to the 'Session History'.

- Optimize brief information displayed in 'Session History'.

- Add information showing compile time and cvs version in About.

- Rename StatusBar and ToolBar to Status Bar and Toolbar.

  - 'Commit' need server capabilitiy `:candidate`.

  - 'Cancel Confirmed Commit' need server capabilitiy `:candidate` and
    `:confirmed-commit`.

    `:confirmed-commit`.

  - 'Discard Changes' need server capabilitiy `:candidate`.
